  1. Introdução

O documento tem por finalidade coletar, analisar e definir necessidades e características gerais do sistema Angélica. Concentra-se nas necessidades apontadas pelo cliente, nas razões que levam a essas necessidades e como elas serão atendidas pelo software.

  1. Escopo

O escopo deste documento se limita a fornecer a todos os envolvidos uma descrição compreensível das funcionalidades que serão atendidas no projeto de software. Quando necessário este documento pode ser atualizado durante todo o ciclo de desenvolvimento da solução.

  1. Objetivos

Este documento deve ser utilizado para uma melhor avaliação no desenvolvimento do software, possibilitando uma melhor noção das necessidades e características que serão descritas no documento de requisitos, para a posterior implementação. Dessa forma, o documento visa esclarecer os objetivos e decisões a serem tomadas na construção do sistema.

  1. Visão Geral do Projeto

Este documento de visão está organizado de forma a fornecer uma ampla visão sobre o escopo do sistema web Angélica. Seu conteúdo está organizado de forma a fornecer uma visão dos envolvidos no projeto, das necessidades apontadas por estes envolvidos, dos benefícios esperados pelos envolvidos, e das funcionalidades e suas características para atender a estes benefícios. Também são apresentados aqui requisitos importantes para o produto, sendo realizado a descrição de cada funcionalidade do sistema.

6.   Posicionamento

6.1 Descrição do Problema

Descrição do problema que deve ser resolvido pelo sistema Angélica:

O problema

Falta de informações de quais motoristas têm permissão em usar um determinado táxi.

Afeta

Diretamente a agência reguladora.

cujo impacto é

Circulação indevida desses veículos.

uma boa solução seria

Implantar um sistema integrado e seguro que auxilie os agentes a regulamentar e fiscalizar os serviços de automóveis de aluguel (táxis).

6.2 Sentença de Posição do Produto

Breve descrição do produto:

Para

Os funcionários da ARSAM, Agência Reguladora dos Serviços Públicos Concedidos do Estado do Amazonas

O Angélica

“Angélica” é um sistema para plataforma web para controle e gerenciamento de motoristas de táxi

Que

Tem o objetivo de informar quais motoristas tem permissão de acesso a um determinado táxi.

Nosso produto

Oferece uma  forma segura e intuitiva para seus usuários.

8.3 Regras de Negócio

8.3.1 - Gerenciar Usuários

        RN[001] - Cadastrar Usuário e Senha

        O cadastro de usuários será feito pelos agentes do órgão solicitante.

RN[002] - Preenchimento Obrigatório

Todos os campos do cadastro de usuários são obrigatórios.

RN[003] - Status do Usuário

Todo usuário cadastrado receberá o Status “Ativo”. Caso não seja mais permitido o acesso deste funcionário ao sistema, ele passará para “Inativo”.

                RN[004] - Exclusão de Cadastro

                O usuário não poderá ser excluído por afetar o conjunto de processos produzidos na relação com os outros usuários. Se houver necessidade de desativação o mesmo deverá receber o status “Inativo”.

        8.3.2 - Logar No Sistema

        RN[005] - Preenchimento Obrigatório

        Deverá obrigatoriamente preencher os campos usuário e senha.

        RN[006] - Validação

        Os dados informados nos campos usuário e senha deverão ser validados junto ao cadastro de usuários no banco de dados.

        8.3.3 - Permissões dos Motoristas

                RN[007] - Motorista proprietário

                Todo táxi têm somente um proprietário.

                RN[008] - Motorista auxiliar

                Todo táxi têm somente um motorista auxiliar.

RN[009] - Permissão do motorista

                Um motorista pode ser proprietário de um táxi e auxiliar de outro.

        8.3.4 - Buscar Informações

                

                RN[010] - Buscar Táxi

                O táxi deverá ser pesquisado através da placa do veículo

                RN[011] - Buscar Motorista

                O motorista deverá ser pesquisado através do seu renach.

                RN[012] - Buscar Usuários

                O usuário deverá ser pesquisado através do seu cpf.
